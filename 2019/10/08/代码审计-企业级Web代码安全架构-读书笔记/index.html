一、通用代码审计思路

  • 敏感函数回溯参数过程

    • 这种方法事实上是我之前复现一些cms时最常用的方法

    • 直接搜索相应的敏感关键字，快速挖掘想要的漏洞。

    • 优点是快

  • 通读全文代码

    • 作者推荐初学者（像我这样的）应该先通读一些小应用（chinaz、admin5等）的全文代码，所以打算有时间通读一波:)，等阅读完一些简单的应用之后，再开始读开源框架（thinkphp、Yii等），天知道这两天复现tp5的反序列化漏洞我有多南。

    • 优点是了解程序的架构以及业务逻辑，能挖掘更多高质量的逻辑漏洞。

  • 根据功能点定向审计
    • 比如文件上传功能、文件管理功能、登录认证功能、找回密码功能等等，我认为相当于和黑盒测试结合，根据一些常出问题的功能点进行审计。
    • 优点也是比较快
